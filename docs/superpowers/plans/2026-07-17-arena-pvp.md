# Arena Mode (PvP) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** FFA PvP deathmatch on a dedicated floor-97 arena with cover, entered from a new "Arena Mode" main-menu row, with full-kit player-vs-player damage, first-to-10 kills, auto-respawn, and a progression firewall.

**Architecture:** The arena is a deterministic sentinel-floor level (`engine_arena.cpp`, cloned from `engine_town.cpp`). PvP damage is a **target registry inside `Combat`** (`s_pvpTargets`) + three geometry helpers (`pvpCone`/`pvpRay`/`pvpRadius`) that each existing entity-damage site calls one line beside its entity query; all hits land through `Combat::applyDamageToPlayer`, so block/perfect-block/invuln work unchanged. Victim HP propagates by the existing remote-view writeback → snapshot path (no new damage wire). Deathmatch state is pure logic in `game/arena.h` (tested), driven by the server, broadcast via three new additive `SV_EVENT`s. `PROTOCOL_VERSION` 19→20.

**Tech Stack:** C++17, doctest, existing engine systems only. New texture (if needed) via `tools/gen_texture.py` — never hand-authored.

**Spec:** `docs/superpowers/specs/2026-07-17-arena-pvp-design.md` (all decisions locked there).

```cpp
#pragma once
// arena.h — pure PvP deathmatch rules for Arena mode (floor 97). Pure functions on plain
// state (the stash.h pattern) so the rules are unit-testable without the engine. The engine
// owns WHEN to call these (server death detection / respawn ticks); this owns the math.
#include "core/types.h"
#include "core/math.h"

namespace Arena {
    constexpr u32 MAX_COMBATANTS = 4;      // == MAX_PLAYERS; static_assert'd at the engine boundary
    constexpr u32 KILL_TARGET    = 10;     // first to 10 wins (v1 fixed; lobby-config parked)
    constexpr f32 RESPAWN_DELAY  = 3.0f;   // seconds dead before auto-respawn

    struct Score { u16 kills[MAX_COMBATANTS] = {}; };

    // Credit a kill. killerSlot 0xFF (environmental/unknown) records nothing. Returns true and
    // sets winnerOut exactly when this kill reaches KILL_TARGET.
    inline bool recordKill(Score& s, u8 killerSlot, u8& winnerOut) {
        if (killerSlot >= MAX_COMBATANTS) return false;
        s.kills[killerSlot]++;
        if (s.kills[killerSlot] >= KILL_TARGET) { winnerOut = killerSlot; return true; }
        return false;
    }

    // Pick the respawn pad whose NEAREST living hostile is farthest away (max-min). Ties keep
    // the lower index — deterministic, so host and any observer agree. No hostiles → pad 0.
    inline u32 farthestPad(const Vec3* pads, u32 padCount, const Vec3* hostiles, u32 hostileCount) {
        u32 best = 0; f32 bestMin = -1.0f;
        for (u32 p = 0; p < padCount; p++) {
            f32 nearest = 1e30f;
            for (u32 h = 0; h < hostileCount; h++) {
                Vec3 d = pads[p] - hostiles[h];
                f32 dsq = d.x*d.x + d.z*d.z;           // XZ plane — pads/players share the floor
                if (dsq < nearest) nearest = dsq;
            }
            if (hostileCount == 0) nearest = 0.0f;
            if (nearest > bestMin) { bestMin = nearest; best = p; }
        }
        return best;
    }
}
```

// Registered once per authoritative tick by the engine when the arena is live; empty in PvE
